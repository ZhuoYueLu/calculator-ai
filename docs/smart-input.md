# 智能输入行为说明

## 🎯 功能概述

计算器现在具有**智能输入行为**，模拟真实计算器的使用体验：

- ✅ 计算完成后输入**数字** → 清空屏幕，开始新计算
- ✅ 计算完成后输入**运算符** → 继续使用当前结果进行运算

## 📱 使用场景

### 场景 1: 开始新计算

```
步骤:
1. 计算: 5 + 3 = 8
2. 显示屏显示: 8
3. 点击数字 9
4. 显示屏清空，显示: 9 ← 开始新计算
```

**效果**: 自动清空，无需手动按 C

### 场景 2: 连续运算

```
步骤:
1. 计算: 5 + 3 = 8
2. 显示屏显示: 8
3. 点击运算符 +
4. 显示屏显示: 8+ ← 继续使用结果
5. 输入 2
6. 点击 =
7. 结果: 10
```

**效果**: 可以继续进行链式计算

### 场景 3: 混合使用

```
示例: 计算 (5+3)×2+10

步骤:
1. 5 + 3 = 8
2. × 2 = 16  ← 使用运算符继续
3. + 10 = 26 ← 使用运算符继续
```

## 💡 详细行为规则

### 规则 1: 数字输入

**触发条件**: 刚完成计算（按下 = 后）

**行为**: 
- 清空当前显示
- 开始新的输入

**示例**:
```
状态: 显示 "42"（刚计算完成）
操作: 点击 "7"
结果: 显示 "7"（清空了42）
```

### 规则 2: 运算符输入

**触发条件**: 刚完成计算（按下 = 后）

**行为**:
- 保留当前结果
- 追加运算符
- 可以继续输入下一个数

**示例**:
```
状态: 显示 "42"（刚计算完成）
操作: 点击 "+"
结果: 显示 "42+"（保留了42）
继续: 输入 "8" → "42+8"
继续: 点击 "=" → "50"
```

### 规则 3: 函数输入

**触发条件**: 刚完成计算

**行为**:
- 清空当前显示
- 开始新的函数输入

**示例**:
```
状态: 显示 "100"（刚计算完成）
操作: 点击 "sin"
结果: 显示 "sin("（清空了100）
```

### 规则 4: 特殊按钮

#### 清除按钮 (C)
- 始终清空所有内容
- 重置为新输入状态

#### 退格按钮 (⌫)
- 如果刚完成计算：清空显示
- 否则：删除最后一个字符

#### 正负切换 (±)
- 对当前显示的数字取反

#### 倒数 (1/x)
- 如果刚完成计算：对结果取倒数
- 否则：对当前表达式取倒数

## ⚙️ 技术实现

### 状态管理

```javascript
const [isNewInput, setIsNewInput] = useState(true);
```

**状态值**:
- `true`: 刚完成计算或刚清空，等待新输入
- `false`: 正在输入中

### 判断逻辑

```javascript
// 定义运算符
const operators = ['+', '-', '*', '/', '^'];
const isOperator = operators.includes(val);

// 刚完成计算 + 输入数字 → 清空
if (isNewInput && !isOperator) {
  setInput(val);
  setIsNewInput(false);
  return;
}

// 刚完成计算 + 输入运算符 → 继续
if (isNewInput && isOperator) {
  setIsNewInput(false);
  setInput(prev => prev + val);
  return;
}
```

### 状态更新时机

1. **设置为 true**（新输入状态）:
   - 按下 `=` 计算完成后
   - 按下 `C` 清除后
   - 页面初始化时

2. **设置为 false**（输入中状态）:
   - 输入任何内容后
   - 开始构建表达式时

## 🚀 最佳实践

### 建议 1: 利用连续运算

```
计算: 100 + 50 + 25 + 10

高效方式:
100 + 50 = 150
+ 25 = 175   ← 直接继续
+ 10 = 185   ← 直接继续
```

### 建议 2: 适时使用清除

```
如果开始完全不同的计算:
1. 按 C 完全清空
2. 开始新计算

而不是依赖自动清空
```

### 建议 3: 注意运算符优先级

```
想要: (5 + 3) × 2
正确: ( → 5 → + → 3 → ) → × → 2 → =
错误: 5 → + → 3 → = → × → 2 → =
      (这会计算 8 × 2，虽然结果相同但逻辑不同)
```
